#1 Attribute typo in Abs payoff — evendates instead of eventdates Critical
sdevpy/montecarlo/payoffs/basic.py · line 352

Why it's bad: self.eventdates = self.subpayoff.evendates references a non-existent attribute. Any call to Abs.set_valuation_date() raises an AttributeError at runtime, silently breaking portfolio valuations that include absolute-value payoffs.

self.eventdates = self.subpayoff.evendates
self.eventdates = self.subpayoff.eventdates

#4 Unsafe array indexing — [0][0] with no existence check Critical
sdevpy/montecarlo/payoffs/basic.py · lines 226, 268, 475

Why it's bad: np.where(eventdates == self.expiry)[0][0] assumes the expiry date is always present in the event-date grid. If it is absent, the result array is empty and [0] raises an IndexError with no meaningful message, making misconfigured trades very hard to debug.

Fix:

self.idx = np.where(eventdates == self.expiry)[0][0]
matches = np.where(eventdates == self.expiry)[0]
if len(matches) == 0:
    raise ValueError(f"Expiry {self.expiry} not found in event date grid")
self.idx = matches[0]

#6 Division by zero in Variance.evaluate() when no historical fixing exists High
sdevpy/montecarlo/payoffs/basic.py · line 434

Why it's bad: spot_paths[:, 0] / self.current_fixing divides by self.current_fixing, which can be None when no historical data is provided. This raises a TypeError (or produces nan silently via NumPy broadcasting), corrupting the variance-swap PV without any warning.

Fix:

if self.current_fixing is not None:
    var = var + spot_paths[:, 0] / self.current_fixing
#7 Division by zero in path_interp_coeffs() with duplicate time-grid entries High
sdevpy/montecarlo/MonteCarloPricer.py · line 81

Why it's bad: w1 = (event_times - t_left) / (t_right - t_left) produces inf or nan if any two adjacent discretisation times are identical. The corrupted weights silently propagate through the entire MC pricing without any error.

Fix:

dt = t_right - t_left
if np.any(dt == 0):
    raise ValueError("Duplicate times detected in discretisation grid")
w1 = (event_times - t_left) / dt
#8 Division by zero in LinearInterpolator.initialize() with duplicate x-values High
sdevpy/maths/interpolation.py · lines 162–163

Why it's bad: Slope computation (y[1] - y[0]) / (x[1] - xl) divides by zero when consecutive x-grid values are identical (e.g. a vol surface with duplicate tenors). The result is inf or nan stored silently in the interpolator, producing wrong values for all subsequent queries.

Fix:

if np.any(np.diff(self.x_grid) == 0):
    raise ValueError("x_grid contains duplicate values")
#9 Debug print() in Variance.evaluate() — executes on every MC call High
sdevpy/montecarlo/payoffs/basic.py · line 436

Why it's bad: print(f"spot paths: {spot_paths.shape}") fires on every simulation path evaluation — potentially thousands of times per price call. This floods stdout, degrades performance, and breaks any pipeline that parses stdout. It indicates the code was never hardened beyond a debugging session.

Fix: Remove the line, or replace with a logging.debug() call:

import logging
log = logging.getLogger(__name__)
log.debug("spot_paths shape: %s", spot_paths.shape)
#10 No type annotations across the entire codebase Medium
Codebase-wide

Why it's bad: Without type hints, IDEs cannot autocomplete correctly, mypy cannot catch type errors before runtime, and callers have no indication what types are expected. In a numerical library this is especially risky — passing a scalar where a NumPy array is expected can produce wrong results instead of an error.

Fix: Add type hints incrementally to public method signatures, starting at the most-used boundaries (Payoff.evaluate, YieldCurve.discount, Interpolation.value). Run mypy in CI to prevent regression.
