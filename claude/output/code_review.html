#1 write_example() filename format doesn't match load_lv_from_folder() — round-trip permanently broken High · Persists from #25
sdevpy/volatility/localvol/localvol_factory.py · line 162 vs line 82

Why it's bad: write_example() at line 162 writes files named "localvol_" + date.strftime("%Y%m%d-%H.%M.%S") + ".json" — with the localvol_ prefix and dots between hours, minutes, and seconds. load_lv_from_folder() at line 82 constructs the expected path as store_date.strftime(dates.DATE_FILE_FORMAT) + ".json" where DATE_FILE_FORMAT = '%Y%m%d-%H%M%S' — no prefix, no dots. Files written by write_example() will never be found by the loader.

Fix: Align the write path with the load path:

file = os.path.join(file, "localvol_" + date.strftime("%Y%m%d-%H.%M.%S") + ".json")
file = os.path.join(file, date.strftime(dates.DATE_FILE_FORMAT) + ".json")
#2 dt.datime typo in calib_lv_black() signature High · New
sdevpy/volatility/localvol/black_calib.py · line 12

Why it's bad: The signature reads dates: list[dt.datime] instead of list[dt.datetime]. def calib_lv_black(surface: ImpliedVol, dates: list[dt.datime], strikes: list[float],

#3 MatrixLocalVol can be serialised but not deserialised via the factory — round-trip broken High · New
sdevpy/volatility/localvol/localvol.py · lines 218–222  ·  sdevpy/volatility/localvol/localvol_factory.py · lines 58–68

Why it's bad: MatrixLocalVol.dump_data() is commented out, so serialisation falls through to TimeInterpolatedLocalVol.dump_data(). That calls section.dump() on each InterpolatedLocalVolSection, producing {'time': ..., 'model': 'cubicspline', 'logm': [...], 'vol': [...]}. But localvol_factory.create_section() dispatches on model.lower() and only knows 'biexp', 'cubicvol', and 'vsvi'. 'cubicspline' falls through to raise ValueError("Unknown section type"). Any MatrixLocalVol saved to disk cannot be loaded back. The same applies to VectorLocalVol whose sections dump model: 'flat'.

Fix: Add 'cubicspline' and 'flat' cases to create_section() that reconstruct InterpolatedLocalVolSection and FlatLocalVolSection respectively — or restore a distinct MatrixLocalVol.dump_data() that uses its own top-level format with a corresponding loader.

#4 Dead assignment section = None immediately before unconditional raise Medium · Persists from #25

Why it's bad: In the case _: branch of create_section(), section = None is assigned and then the function immediately raises ValueError.         case _:
            section = None
            raise ValueError(f"Unknown section type: {model}")
#5 __main__ test script in localvol_factory.py constructs wrong JSON structure Medium · New
sdevpy/volatility/localvol/localvol_factory.py · lines 206–215

Why it's bad: lv_data = lv.dump_data() returns {'name': ..., 'valdate': ..., 'snapdate': ..., 'sections': [...]}. The script then wraps the entire dict as the sections value: data = {'valdate': ..., 'snapdate': ..., 'sections': lv_data}. When this JSON is loaded by load_lv_from_data(), data['sections'] is the full object dict instead of the section list, so the subsequent for section_config in sections loop iterates over dict keys ('name', 'valdate', …) and raises a confusing KeyError or TypeError with no clear cause.

Fix: Use lv_data directly — it already contains the complete structure:

data = {'valdate': valdate.strftime(dates.DATE_FORMAT),
        'snapdate': valdate.strftime(dates.DATETIME_FORMAT),
        'sections': lv_data}
data = lv_data  # dump_data() already includes name/valdate/snapdate/sections
