#1 shift_forward returns None when a shift is applied — silently corrupts every subsequent PDE step Critical
sdevpy/pde/forwardpde.py · lines 101–116

Why it's bad: The if branch (when the forward is off by more than tol) computes p_shifted correctly but never returns it — only the else branch has a return p. When a shift is actually needed, the function falls through and implicitly returns None. In density_step, p = shift_forward(old_x, p) then assigns None to p, after which mass = np.trapezoid(p, x) raises TypeError. The bug is currently latent because PdeConfig.shift_forward defaults to False, but it will fire the moment the feature is turned on. 

if np.abs(pde_forward_m - target_forward_m) > tol:
    ...
    p_shifted = np.maximum(p_shifted, 0.0)
    pde_forward_m = np.trapezoid(ex * p_shifted, x)
    print(...)
    # ← missing return p_shifted
else:
    return p  # only this branch returns a value

Fix: Add the missing return at the end of the if block:

    pde_forward_m = np.trapezoid(ex * p_shifted, x)
    print(...)
    return p_shifted
else:
    return p
#2 check_degrees_of_freedom boolean attribute shadows the method of the same name — TypeError on every calibration call Critical
sdevpy/models/surfaces/zerosurface.py · lines 24, 218–219, 224

Why it's bad: __init__ sets self.check_degrees_of_freedom = True, an instance attribute that shadows the method def check_degrees_of_freedom(self, options) defined further down. In check_consistency(): 

if self.check_degrees_of_freedom:        # truthy boolean — OK
    self.check_degrees_of_freedom(c_options) # calls True(...) → TypeError!

Python raises TypeError: 'bool' object is not callable on every calibration call where the flag is True. This is the same attribute/method shadowing pattern that previously affected expiry_times and was flagged in the last review.

Fix: Rename the boolean flag:

self.check_degrees_of_freedom = True
self.do_check_degrees_of_freedom = True

if self.check_degrees_of_freedom:
    self.check_degrees_of_freedom(c_options)
if self.do_check_degrees_of_freedom:
    self.check_degrees_of_freedom(c_options)
#3 Debug print() calls in shift_forward fire on every PDE step when forward-shifting is enabled High
sdevpy/pde/forwardpde.py · lines 105, 114

Why it's bad: Two unconditional print() calls inside shift_forward will flood stdout during calibration or MC pricing the moment config.shift_forward = True is set, producing thousands of lines of output, breaking any output-parsing pipeline, and degrading performance through repeated console flushing. 

print(f"PDE forward moneyness: {pde_forward_m}")
print(f"After shift: {pde_forward_m}")

Fix: Remove both lines. If diagnostics are needed, use logging.debug().

#5 Flat-vol sigma fixed at TEST_VOL = 0.20 — not injectable via constructor High
sdevpy/montecarlo/assetmodels.py · lines 5, 25

Why it's bad: When use_lv=False (the default), every asset is simulated at exactly 20% flat volatility regardless of market data. The name TEST_VOL signals testing intent, yet no kwargs hook exists to override it. Any production use of the constant-vol path yields wrong prices silently.

Fix:

self.sigma = np.asarray([TEST_VOL for _ in range(self.n_factors)])
sigma = kwargs.get('sigma', [TEST_VOL] * self.n_factors)
self.sigma = np.asarray(sigma)

#10 No tests for shift_forward, calibration path, or TsSvi1 evaluation Medium
sdevpy/tests/ · codebase-wide

Why it's bad: Issue #1 (missing return in shift_forward) and issue #2 (check_degrees_of_freedom shadow) would both be caught immediately by basic smoke tests. The pattern is consistent: critical bugs are introduced in untested code and survive until manual end-to-end runs happen to exercise the path. TsSvi1.formula(), convert_option(), convert_target(), and shift_forward all remain entirely unverified.

Fix: Add at minimum: (1) a test_shift_forward that sets config.shift_forward = True and asserts the returned density is not None and integrates to 1; (2) a ZeroSurface.check_consistency() smoke test to catch the recurring attribute/method shadow bug class; (3) a TsSvi1 forward-evaluation test checking output shape and positivity.
