#2 gsvi_check_params() is a stub — always returns valid High
sdevpy/volatility/impliedvol/models/gsvi.py · line 43

Why it's bad: gsvi_check_params unconditionally returns True, 0.0. TsSvi1.check_params() calls it to decide whether to return a calibration penalty, so the penalty branch in TsSvi1ObjectiveBuilder.objective() is dead code. The optimizer will evaluate the formula on infeasible parameter sets — producing negative variance and a ValueError — instead of receiving a smooth penalty that guides it back into the feasible region. The real constraint logic already exists in svi_check_params() and can be adapted directly.

Fix (minimum viable):

def gsvi_check_params(params, check_butterfly=False):
    a, b, rho, m, sigma = params[0], params[1], params[2], params[3], params[4]
    if np.any(b < 0) or np.any(np.abs(rho) >= 1) or np.any(sigma < 0):
        return False, constants.FLOAT_INFTY
    if np.any(a + b * sigma * np.sqrt(1 - rho**2) < 0):
        return False, constants.FLOAT_INFTY
    return True, 0.0

This constraint is what gsvi_check_params should be implementing but isn't. The question has been answered by the code structure (the penalty mechanism exists), but the implementation is missing.
