########## Claude's recommendations on Architecture ###############################################
1. Use the existing MC framework consistently

fbsabr.py, mcheston.py, and mczabr.py each hand-roll their own Monte Carlo loop
(draw gaussians → evolve → collect payoff), duplicating the logic that already lives in
PathGenerator. The architecture has the right abstractions — Payoff, PathGenerator, MarketState —
but the analytics modules bypass them entirely. Unifying them would eliminate the duplication and
make the analytics modules benefit from improvements to the MC engine.

2. Define protocols for market data boundaries

eqvolsurface.eqvolsurfacedata_from_file() returns an object whose shape is implicit — consumers
must know that it has .expiries, .forwards, .vols, .get_strikes(). A typing.Protocol
(or a @dataclass) would make the contract explicit and allow static analysis to catch misuse.
The same applies to the parameters dictionaries passed to the MC pricers
({'LnVol': ..., 'Beta': ..., 'Rho': ...}) — these should be typed dataclasses, not plain dicts.

3. Extract price_straddles() out of bachelier.py

It is a matrix-level utility, not an atomic pricing function. It breaks the module's single
responsibility and has a completely different interface from everything else in the file. It belongs
in a higher-level utility module (e.g., sdevpy/analytics/utils.py or alongside the straddle-specific
project code).

4. Introduce a shared protocol for analytics modules

black.py and bachelier.py expose the same logical interface — price, implied_vol — but there's no
formal contract. A Protocol class would let calibration code, zerosurface.py, and tests reference a
model type rather than a concrete module:

class AnalyticsModel(Protocol):
    def price(self, expiry, strike, is_call, fwd, vol): ...
    def implied_vol(self, expiry, strike, is_call, fwd, fwd_price): ...

5. Replace the repeated match calculate_type blocks in zerosurface.py

forward_price(), black_volatility(), bachelier_volatility(), and shifted_black_volatility() all contain
the same four-case match on self.calculate_type. Every time a new OptionQuoteType is added, all four
methods must be updated. A dispatch dict defined once would eliminate the duplication and make the
open/closed principle hold.

The inconsistent return type of implied_vol_newton() (issue #1 from the last review) also counts against
Architecture — a function that returns either a scalar or an array depending on input size is not a
well-defined contract.
